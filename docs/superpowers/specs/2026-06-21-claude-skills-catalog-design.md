# Дизайн: каталог скиллов для Claude Code (`claude-skills`)

Дата: 2026-06-21
Статус: утверждён владельцем, готов к плану реализации

## 1. Зачем (цель)

Публичный GitHub-репозиторий — **каталог полезных скиллов/плагинов для Claude Code**.
Решает две задачи:
- **для себя** — собрать в одном месте годные скиллы и помнить, чем каждый хорош;
- **делиться** — чтобы другой человек зашёл, прочитал список и поставил себе понравившийся.

Ключевой принцип: **ссылки, а не копии**. Чужой код не копируется в репозиторий — хранится
только карточка-описание со ссылкой на оригинал.

## 2. Решения (зафиксировано в брейншторме)

| Вопрос | Решение |
|---|---|
| Что умеет репо | Гибрид: читаемый каталог + установка одной командой |
| Единица каталога | Плагин/репозиторий (как `obra/superpowers` = 1 плагин, 14 скиллов внутри) |
| Ссылка vs копия | Только ссылка на внешний репо + зафиксированный коммит |
| Проверка безопасности | Ручное ревью по чек-листу при приёме PR + пин на проверенный коммит |
| Масштаб | Личный + пара контрибьюторов → минимум автоматики |
| Дом | `hahpiez4-source/claude-skills`, публичный |

## 3. Архитектура

Единственный «источник правды» — карточки в `entries/`. Из них скрипт-сборщик `build.py`
генерирует читаемый `README.md` и технический `marketplace.json`. Руками генерируемые файлы
не редактируются — это исключает рассинхрон.

```
claude-skills/
├─ README.md                      # ГЕНЕРИРУЕТСЯ: витрина-таблица + как ставить
├─ .claude-plugin/
│  └─ marketplace.json            # ГЕНЕРИРУЕТСЯ: для `claude plugin marketplace add`
├─ entries/                       # ★ ИСТОЧНИК ПРАВДЫ — одна карточка на скилл
│  ├─ _template.md                # шаблон для контрибьюторов
│  └─ superpowers.md
├─ build.py                       # генератор: entries/*.md → marketplace.json + README
├─ CONTRIBUTING.md                # как добавить скилл + правила
├─ docs/security-checklist.md     # на что смотреть (prompt injection)
└─ .github/PULL_REQUEST_TEMPLATE.md   # чек-лист в каждом PR
```

## 4. Формат карточки (`entries/*.md`)

Frontmatter (машиночитаемая часть) + свободный текст ниже.

```yaml
---
name: superpowers
source: obra/superpowers           # github owner/repo — ССЫЛКА, не копия
pin: <commit-sha>                  # зафиксированный проверенный коммит
category: development              # development / content / seo / utils ...
tags: [tdd, debugging, planning]
why: "Методология разработки: дизайн → план → TDD → ревью."
usecases:
  - "Нужна дисциплина TDD и системная отладка"
  - "Нужен структурированный процесс от идеи до кода"
security:
  reviewed_by: hahpiez4-source
  reviewed_commit: <sha>
  date: 2026-06-21
  prompt_injection: clean          # clean / flagged
  notes: "Прочитал все SKILL.md, инъекций не найдено."
---

# Superpowers
Свободный текст: подробности, заметки, ссылки.
```

Обязательные поля (проверяются сборщиком): `name`, `source`, `pin`, `why`,
`security.reviewed_by`, `security.prompt_injection`.

Поля `pin` + `security` закрывают риск «автор подменит содержимое после проверки»:
фиксируется ровно тот коммит, который проверяли.

## 5. Сборщик `build.py` (поток данных)

1. Читает все `entries/*.md` (кроме `_template.md`).
2. Валидирует: обязательные поля заполнены; `name` уникальны; `pin` — это коммит, не ветка.
   При ошибке — внятное сообщение с именем файла, сборка падает.
3. Генерирует:
   - `.claude-plugin/marketplace.json` — список плагинов (name, source github, описание; пин в
     `source`, если формат маркетплейса это поддерживает — проверить при реализации);
   - `README.md` — таблица-витрина (название, категория, зачем, юзкейсы, статус проверки, ссылка)
     + инструкция по установке.
4. Запускается локально перед коммитом. Режим `--check` (сборка идемпотентна: повторный запуск
   не даёт diff) — пригодится для опциональной CI-проверки позже.

## 6. Флоу контрибьютора + проверка безопасности

**Добавление скилла:**
1. Копирует `entries/_template.md`, заполняет карточку.
2. Запускает `build.py` (README и marketplace.json пересобираются).
3. Открывает Pull Request.

**Шаблон PR (чек-лист):**
- [ ] Карточка заполнена по шаблону
- [ ] Указан конкретный проверенный коммит (`pin`), не ветка
- [ ] Прочитал содержимое скилла, prompt injection не найдено

**Ревью перед мерджем (мейнтейнер):**
Открыть оригинал скилла на коммите `pin` и пройти `docs/security-checklist.md`:
скрытые инструкции (exfiltration, удаление файлов, «ignore previous instructions»),
обращения к неизвестным хостам, запрос лишних прав/инструментов, обфускация (base64).
Чисто → принять. Подозрительно → отклонить.

## 7. Проверка/тесты

- Валидация в `build.py` — основной механизм качества.
- Идемпотентность: `build.py --check` подтверждает, что закоммиченные сгенерированные файлы
  совпадают со сборкой из карточек.
- Масштаб личный → тяжёлый CI не нужен; при росте можно добавить GitHub Action на `--check`.

## 8. Открытые вопросы к реализации

- Точный формат `marketplace.json` в Claude Code: поддерживает ли `source` пин на коммит/тег.
  Если да — писать туда; если нет — `pin` остаётся гарантией в карточке и в инструкции установки.
- Язык генерируемого README: RU/EN или оба (решить при реализации).
